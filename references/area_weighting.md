# 身体面积占比参考

> 用于色彩评分时的面积加权。不同衣物类型占身体可视面积的比例不同。

## 成年男性身体面积占比（站立正面视角）

| 部位 | 面积占比 | 说明 |
|------|---------|------|
| 头部 | 7% | 含头发、脸部 |
| 帽子 | 3-5% | 棒球帽~3%，渔夫帽~5% |
| 围巾 | 3-5% | 围在脖子上~3%，披肩~5% |
| 内衬/打底 | 15-20% | T恤、衬衫（可见部分） |
| 外套/夹克 | 25-30% | 主要视觉面积 |
| 裤子 | 25-30% | 长裤 |
| 鞋子 | 5-8% | 运动鞋~5%，靴子~8% |
| 袜子 | 1-2% | 通常不可见 |
| 手套 | 2-3% | 手部面积 |
| 包/配饰 | 3-5% | 包包、腰带、手表等 |

## 面积对色彩感知的影响

### 规律

1. **大面积颜色主导视觉**
   - 外套/裤子的颜色决定了整体色调
   - 配饰颜色是点缀

2. **面积比例影响和谐度**
   - 主色 60% + 辅色 30% + 点缀色 10% → 经典比例
   - 50% + 50% → 容易显得割裂

3. **颜色互换效果不同**
   - 红外套 + 黑裤子 vs 黑外套 + 红裤子
   - 前者：红色主导，更张扬
   - 后者：黑色主导，红色点缀，更稳重

### 评分建议

```
当前：extract_features([(hex1), (hex2), (hex3)])
      → 等权重，每个颜色占 33%

改进：extract_features([(hex1, 0.30), (hex2, 0.18), (hex3, 0.06)])
      → 面积加权，外套占 30%，T恤占 18%，鞋子占 6%
```

## 实现路径

### 方案 A：纯知识驱动（推荐先做）

```python
# 1. 读取衣物类型
item_type = item["type"]  # "外套", "T恤", ...

# 2. 查表获取面积
area = type_area_map.get(item_type, 0.1)  # 默认 10%

# 3. 传入特征提取
features = extract_features([(hex1, area1), (hex2, area2), ...])
```

优点：不需要重新训练模型，立即可用

### 方案 B：Agent 识别面积（进阶）

```
用户上传照片 → Agent 识别衣物类型 + 估算面积占比
  → "这件外套占身体面积约 30%"
  → 保存到 YAML
```

优点：更准确，能处理叠穿、特殊版型

### 方案 C：重新训练模型（最准）

```
收集数据：
  - 穿搭照片 + 人类评分
  - 标注每件衣物的面积占比

训练：
  - 特征 = 颜色 + 面积
  - 标签 = 人类评分
```

缺点：需要大量标注数据，成本高

---

## 建议实施顺序

1. **立即做**：方案 A — 创建面积映射表，修改评分逻辑
2. **短期做**：Agent 识别衣物类型时，同时估算面积
3. **长期做**：如果需要更准确，再考虑重新训练

这样可以快速验证面积因素的效果，再决定是否值得投入更多资源。
