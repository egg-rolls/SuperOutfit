# SuperOutfit — AI 智能穿搭顾问

> 用 AI 帮每个人穿得好看，买得聪明。

---

## 一句话介绍

SuperOutfit 是一个 **AI 驱动的个人衣橱管理系统**，通过色彩和谐度模型、穿着追踪和智能推荐，帮助用户最大化利用现有衣物、减少冲动消费、提升穿搭品质。

---

## 解决什么问题

| 痛点 | 现状 | SuperOutfit 方案 |
|------|------|-----------------|
| **衣橱利用率低** | 80% 的衣物每月穿不到 1 次 | 穿着追踪 + 闲置分析，找出"买了不穿"的衣服 |
| **买衣服凭感觉** | 不知道缺什么，容易重复购买 | 购物清单 + 衣橱缺口分析，按需购买 |
| **搭配靠灵感** | 不确定颜色搭不搭、风格统不统一 | 5462 组真实色卡训练的色彩评分模型 |
| **换季手忙脚乱** | 不知道哪些该收、哪些该拿出来 | AI 根据天气和衣物数据自动建议换季整理 |
| **衣物寿命不清晰** | 不知道穿了多少次、性价比如何 | 穿着次数追踪 + 单次穿着成本计算 |

## 核心能力

### 🎨 色彩和谐度评分

基于 **5462 组真实人类审美色卡**训练的高斯过程模型。

不是拍脑袋的规则，是 **4 万人用点赞投票出来的审美共识**。

```
SSS ≥ 85   顶级     名画级配色（前 8%）
SS  ≥ 75   优秀     专业搭配师水准
S   ≥ 65   出色     有明确审美风格
A   ≥ 50   良好     和谐不出错
B   ≥ 35   一般     中规中矩
C   < 35   需改进   建议调整
```

**技术栈：** 42 维特征（HSL + OKLab）→ 点赞迁移学习 → 稀疏高斯过程（500 诱导点）

**数据来源：** Color Hunt + 色采（名画/故宫/摄影配色）+ LOL Colors + Design Seeds + Colormind

**反向推导：** 给定已知颜色和目标分数，AI 自动推荐缺失颜色。

### 🧥 衣橱管理

每件衣物一个 YAML 文件 + 一张图片。

- **AI 写 YAML → 系统处理**：不用填表，AI 分析图片后直接生成结构化数据
- **统一 CRUD**：衣柜和购物清单用同一套命令，一个 flag 切换
- **批量操作**：按类型、按季节批量管理
- **数据自包含**：穿着历史、清洗记录全在衣物自己的文件里

### 📊 穿着追踪

- 记录每天穿了什么（支持多件批量记录）
- 自动计算"距离上次清洗穿了几次"
- 超过建议清洗次数自动提醒
- 穿着报表：最常穿 Top 5、性价比排行、闲置衣物

**性价比公式：** `单次穿着成本 = 购入价格 ÷ 穿着次数`

→ 帮助用户发现"贵但穿得多 = 真便宜"的衣物。

### 🛒 智能购物清单

- 用户说"想买裙子" → AI 添加到购物清单
- 基于现有衣物穿着数据动态计算推荐率
- 同类型穿得多 → 高推荐（说明你需要）
- 同类型闲置 → 低推荐（说明你不缺）

### 🌤️ 天气集成

- 接入实时天气 API
- 根据温度推荐适合的衣物
- 大连等沿海城市支持经纬度精确定位

## 技术架构

```
┌─────────────────────────────────────────────┐
│              SuperOutfit CLI                 │
│         (AI Agent 直接调用的接口)             │
├─────────────────────────────────────────────┤
│  spof add/list/show/edit/delete             │  CRUD
│  spof wear add/wash/check/report            │  穿着管理
│  spof color score/inverse                   │  色彩分析
│  spof weather / export / import             │  辅助工具
├─────────────────────────────────────────────┤
│              YAML 数据层                     │
│  data/items/*.yaml    衣物数据               │
│  data/wishlist/*.yaml 购物清单               │
│  data/profile.yaml    用户画像               │
└─────────────────────────────────────────────┘
         ↕                    ↕
┌────────────────┐  ┌─────────────────────┐
│   MCP Server   │  │   Vue 前端 + API    │
│  (AI Agent 用) │  │   (Web 界面)        │
└────────────────┘  └─────────────────────┘
```

### 技术栈

| 层 | 技术 |
|----|------|
| CLI | Python 3.11 + argparse |
| 后端 | FastAPI + uvicorn |
| 前端 | Vue 3 + Vite + Naive UI |
| AI 模型 | scikit-learn 稀疏高斯过程 |
| 数据 | YAML 文件（用户可直接编辑） |
| 依赖管理 | uv (Python) + npm (Node) |
| AI 集成 | MCP 协议（兼容 Claude Code / Cursor / OpenCode） |

### 设计原则

1. **工具做 CRUD，AI 做决策** — 不在代码里硬编码业务逻辑
2. **数据自包含** — 每件衣物的所有数据在一个 YAML 文件里
3. **CLI 优先** — 命令行是主要接口，Web 是辅助
4. **AI 原生** — 命令设计为 AI 容易调用的格式（JSON 输出、批量操作）

## 用户画像

### 目标用户

- **大学生 / 年轻职场人**：预算有限，想穿得好看但不想花太多钱
- **极简主义者**：想减少衣物数量，提高利用率
- **穿搭爱好者**：想系统化管理衣橱，提升搭配水平
- **AI 早期采用者**：喜欢用 AI 工具优化日常生活

### 用户场景

1. **出门前**："今天穿什么？" → AI 根据天气 + 衣橱推荐
2. **购物时**："我需要买这个吗？" → 查看购物清单和缺口分析
3. **换季时**："哪些该收起来？" → AI 按季节标签建议整理
4. **月底**："这个月穿了什么？" → 穿着报表 + 性价比分析
