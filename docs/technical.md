## 1. 色彩空间：从 RGB 到 OKLab

### 什么是色彩空间？

色彩空间就是"用数字表示颜色的方式"。常见的有：

| 色彩空间 | 表示方式 | 特点 |
|---------|---------|------|
| RGB | (红, 绿, 蓝) | 电脑屏幕用，但不符合人眼感知 |
| HSL | (色相, 饱和度, 亮度) | 符合直觉，但颜色距离不准确 |
| OKLab | (亮度, 绿红轴, 蓝黄轴) | **符合人眼感知，等距离 = 等差异** |

### 为什么不用 RGB？

RGB 的问题是：**数字上的距离 ≠ 人眼感受到的差异**。

```
例子：
  红色 (255, 0, 0) 和 深红 (200, 0, 0)
  数字差：55
  人眼感受：差异不大

  绿色 (0, 255, 0) 和 深绿 (0, 200, 0)
  数字差：55
  人眼感受：差异也很不大

  但是！
  红色 (255, 0, 0) 和 绿色 (0, 255, 0)
  数字差：很大
  人眼感受：差异巨大！
```

RGB 把这三个差异当成一样大，但人眼不是这样工作的。

### 为什么不用 HSL？

HSL 比 RGB 好，但还有问题：

```
例子：
  浅黄 (H=60, S=100, L=50) 和 浅蓝 (H=240, S=100, L=50)
  H 差了 180°，看起来差异很大

  浅黄 (H=60, S=100, L=50) 和 浅黄绿 (H=90, S=100, L=50)
  H 只差 30°，看起来差异也挺大

  但是！
  深黄 (H=60, S=100, L=20) 和 深蓝 (H=240, S=100, L=20)
  H 也差 180°，但因为都很暗，看起来差异没那么大
```

HSL 的色相（H）在不同亮度下，人眼感知到的"距离"不一样。

### OKLab 的优势

OKLab 是 2020 年才提出的新色彩空间，专门为"符合人眼感知"设计：

```
OKLab 的三个轴：
  L = 亮度 (Lightness)
  a = 绿 ↔ 红 轴
  b = 蓝 ↔ 黄 轴

核心优势：
  在 OKLab 空间中，两点之间的距离 = 人眼感受到的颜色差异
  无论颜色是深是浅，是红是蓝，距离都准确
```

**类比**：
- RGB 就像用"步数"衡量距离，但上坡 100 步和下坡 100 步的体力消耗完全不同
- OKLab 就像用"体力消耗"衡量距离，上坡下坡都换算成实际体力

### 代码实现

```python
def hex_to_oklab(hex_str):
    # 1. HEX → RGB (0-255)
    r, g, b = hex_to_rgb(hex_str)
    
    # 2. RGB → 线性 RGB (去掉 gamma 校正)
    #    显示器为了让人眼看起来均匀，会把数值做非线性变换
    #    我们要反向操作，得到"真实"的光强度
    rl = srgb_to_linear(r)
    gl = srgb_to_linear(g)
    bl = srgb_to_linear(b)
    
    # 3. 线性 RGB → LMS (人眼三种锥体细胞的响应)
    #    人眼有三种感光细胞：长波(红)、中波(绿)、短波(蓝)
    #    这一步把 RGB 转换成这三种细胞的响应值
    l = 0.4122214708*rl + 0.5363325363*gl + 0.0514459929*bl
    m = 0.2119034982*rl + 0.6806995451*gl + 0.1073969566*bl
    s = 0.0883024619*rl + 0.2817188376*gl + 0.6299787005*bl
    
    # 4. LMS → OKLab (立方根压缩 + 线性变换)
    #    人眼对亮度的感知是对数关系（暗处更敏感）
    #    立方根模拟了这种压缩
    L = 0.2104542553*l**0.333 + 0.7936177850*m**0.333 - 0.0040720468*s**0.333
    a = 1.9779984951*l**0.333 - 2.4285922050*m**0.333 + 0.4505937099*s**0.333
    b = 0.0259040371*l**0.333 + 0.7827717662*m**0.333 - 0.8086757660*s**0.333
    
    return L, a, b
```

## 2. 特征提取：怎么把颜色变成数字

### 问题

机器学习模型需要数字输入，但颜色是"概念"。我们需要把一组颜色转换成一个数字向量（特征）。

### 42 维特征

SuperOutfit 用 42 个数字描述一组颜色：

```
HSL 特征 (27 维)：
├── 加权平均 (3维): 色相、饱和度、亮度的加权平均
├── 简单平均 (3维): 色相、饱和度、亮度的简单平均
├── 色相统计 (4维): 最大、最小、范围、标准差
├── 饱和度统计 (4维): 最大、最小、范围、标准差
├── 亮度统计 (4维): 最大、最小、范围、标准差
├── 色相距离 (4维): 平均、最大、最小、标准差
├── 有彩色比例 (1维): 非黑白灰的颜色占比
├── 面积熵 (1维): 颜色分布均匀度
└── 其他 (3维): 颜色数量、亮度差、饱和度差

OKLab 特征 (15 维)：
├── 加权平均 (3维): L、a、b 的加权平均
├── 简单平均 (3维): L、a、b 的简单平均
├── 亮度统计 (4维): 最大、最小、范围、标准差
├── 色度统计 (4维): 最大、最小、范围、标准差
└── 色相距离 (1维): OKLab 色相的平均距离
```

### 为什么需要 42 维？

每个维度捕捉颜色的某个"方面"：

```
例子：两组颜色

色卡 A: 红 + 蓝 (互补色)
色卡 B: 红 + 橙 (类似色)

HSL 色相距离：
  A: 180° (最大)
  B: 30° (较小)

OKLab 色度统计：
  A: 色度差异大 (红是高色度，蓝是中色度)
  B: 色度差异小 (都是高色度)

面积熵：
  A: 如果红蓝各占 50%，熵最大
  B: 如果红 80% 橙 20%，熵较小
```

通过 42 个维度，模型能"看到"颜色组合的各种特征，而不仅仅是"红+蓝"。

### 类比

就像描述一个人：
- 只说"身高 180cm" → 信息太少
- 说"身高 180cm，体重 70kg，年龄 25，戴眼镜，短发..." → 信息丰富

42 维特征就是给颜色组合做一次"全面体检"。

## 3. 高斯过程 (GP)：会"猜"的模型

### 什么是 GP？

高斯过程是一种"会猜"的机器学习模型，特别适合小数据集。

**类比**：
- 神经网络就像一个"死记硬背"的学生，需要大量题目才能学会
- GP 就像一个"举一反三"的学生，看几道题就能猜出规律

### GP 的工作原理

GP 的核心思想：**相似的输入 → 相似的输出**

```
训练数据：
  色卡 A (特征 X₁) → 分数 85
  色卡 B (特征 X₂) → 分数 60

新色卡 C (特征 X₃)：
  如果 X₃ 和 X₁ 很像 → 猜分数 ≈ 85
  如果 X₃ 和 X₂ 很像 → 猜分数 ≈ 60
  如果 X₃ 和两者都不像 → 猜分数 ≈ 72（不确定）
```

GP 不仅猜分数，还会告诉你"我有多确定"：

```
预测结果：
  分数 = 72 ± 15（不确定）
  分数 = 85 ± 2（很确定）
```

### 为什么选择 GP？

| 模型 | 优点 | 缺点 | 适合场景 |
|------|------|------|---------|
| 线性回归 | 简单快速 | 只能学线性关系 | 数据量大，关系简单 |
| 神经网络 | 能学复杂关系 | 需要大量数据 | 数据量 > 10000 |
| **GP** | **小数据也能学，输出不确定性** | 训练慢 O(n³) | **数据量 < 10000** |

SuperOutfit 的色卡数据只有 5462 组，GP 是最佳选择。

### GP 的数学（简化版）

```
GP 假设：
  任意 N 个点的函数值服从 N 维高斯分布

  f(X₁, X₂, ..., Xₙ) ~ N(μ, K)

  其中：
    μ = 均值（通常设为 0）
    K = 协方差矩阵（核函数计算）

核函数 K(Xᵢ, Xⱼ)：
  衡量 Xᵢ 和 Xⱼ 的"相似度"
  越相似 → 协方差越大 → 输出越接近
```

**Matérn 5/2 核**（SuperOutfit 使用）：

```
K(Xᵢ, Xⱼ) = σ² (1 + √5·r + 5/3·r²) · exp(-√5·r)

其中 r = ||Xᵢ - Xⱼ|| / l

参数：
  σ² = 信号方差（控制输出范围）
  l = 长度尺度（控制"多远算远"）
```

## 4. 稀疏 GP：让 GP 跑得更快

### 问题

标准 GP 的训练复杂度是 O(n³)，n 是数据量。

```
5462 组色卡：
  训练时间 ≈ 5462³ ≈ 1630 亿次运算
  实际时间 ≈ 5-10 分钟
```

### 解决方案：稀疏 GP

核心思想：**不用全部数据，只选 500 个"代表"**

```
原始数据：5462 组色卡
  ↓
K-Means 聚类：分成 500 簇
  ↓
选 500 个诱导点：每个簇选最靠近中心的点
  ↓
用 500 个诱导点训练 GP
```

### 为什么能工作？

K-Means 聚类会把相似的色卡分到同一簇：

```
簇 1：暖色调色卡（红/橙/黄）
簇 2：冷色调色卡（蓝/绿/紫）
簇 3：中性色卡（灰/白/黑）
...
```

每个簇选一个代表，就覆盖了所有"颜色类型"。

### 复杂度对比

```
标准 GP：O(n³) = O(5462³) ≈ 1630 亿
稀疏 GP：O(n·m²) = O(5462·500²) ≈ 13.6 亿

加速比：120 倍
实际训练时间：5-10 分钟 → 10 秒
```

### 精度损失

稀疏 GP 会损失一些精度，但通常很小：

```
标准 GP MAE：~5.0
稀疏 GP MAE：~6.95

损失：~2 分（在 0-100 分的尺度上，可以接受）
```

## 5. 点赞迁移：从点赞数到分数

### 问题

色卡数据来自多个网站：
- Color Hunt：有点赞数（41310 likes, 38169 likes...）
- 色采：没有点赞数
- LOL Colors：没有点赞数

如何给没有点赞数的色卡打分？

### 解决方案：点赞迁移

**第一步：把点赞数转换成百分位分数**

```
点赞数分布（高度偏斜）：
  大部分色卡：0-100 likes
  少数色卡：10000+ likes

直接用点赞数当分数？不行！
  100 likes 的色卡可能比 10 likes 的好 10 倍吗？
  不一定，可能只是发布时间更长

解决方案：用百分位
  100 likes 的色卡，如果超过了 90% 的色卡 → 分数 90
  10 likes 的色卡，如果超过了 30% 的色卡 → 分数 30
```

**第二步：用有赞色卡训练 GP**

```
训练数据：
  特征 X（42 维） → 分数 y（百分位）

  X₁ (暖色调, 3色, 高对比) → 90
  X₂ (冷色调, 2色, 低对比) → 40
  ...
```

**第三步：用 GP 预测无赞色卡的分数**

```
新色卡（无赞）：
  特征 X_new → GP 预测 → 分数 65
```

### 类比

就像考试：
- 有赞色卡 = 已经知道分数的学生
- 无赞色卡 = 还没考试的学生
- GP = 根据"学习特征"（42 维）预测考试分数

## 6. 评分等级：SSS~D 是怎么定的

### 分数分布

5462 组色卡的分数分布：

```
统计：
  P50 (中位数) = 63.4
  P75 = 74
  P90 = 82
  P95 = 87
  P99 = 96
```

### 等级划分

根据分布，划分 7 个等级：

```
SSS ≥ 85   前 ~8%    顶级，名画级配色
SS  ≥ 75   前 ~20%   优秀，专业搭配师水准
S   ≥ 65   前 ~35%   出色，有明确审美风格
A   ≥ 50   前 ~55%   良好，和谐不出错
B   ≥ 35   前 ~80%   一般，中规中矩
C   ≥ 20   前 ~93%   偏弱，需要调整
D   < 20   后 ~7%    差，明显不搭
```

### 为什么用百分位？

```
如果用绝对分数：
  90 分 = 好？但可能 90% 的色卡都超过 90 分
  50 分 = 差？但可能 50 分已经超过了 80% 的色卡

用百分位：
  90 分 = 超过了 90% 的色卡 → 真的好
  50 分 = 超过了 50% 的色卡 → 真的中等
```

### 等级含义

| 等级 | 含义 | 使用建议 |
|------|------|---------|
| SSS | 顶级配色 | 直接用，不用改 |
| SS | 优秀 | 可以直接用 |
| S | 出色 | 可以用，微调更好 |
| A | 良好 | 可以用，但建议优化 |
| B | 一般 | 建议调整 |
| C | 偏弱 | 需要较大调整 |
| D | 差 | 建议换掉 |

## 总结

SuperOutfit 的色彩评分系统：

```
输入：一组 HEX 颜色
  ↓
特征提取：42 维向量 (27 HSL + 15 OKLab)
  ↓
GP 预测：0-100 分
  ↓
等级划分：SSS/SS/S/A/B/C/D
```

关键设计决策：

1. **OKLab 替代 RGB/HSL** — 等距离 = 等感知差异
2. **42 维特征** — 全面捕捉颜色组合的各种特征
3. **GP 替代神经网络** — 小数据也能学，输出不确定性
4. **稀疏 GP** — 500 诱导点，训练快 120 倍
5. **点赞迁移** — 从人类点赞学习审美共识
6. **百分位等级** — 相对排名，避免绝对分数的误导
